# 숫자 야구게임
이 프로젝트는 Unreal Engine 5를 사용하여 데디케이티드 서버 구조에서 동작하는 멀티플레이어 숫자야구 게임을 구현한 예제입니다.

플레이어는 채팅을 통해 3자리 숫자를 입력하고, 서버가 이를 검증하여 Strike, Ball, Out 결과를 반환합니다.

모든 플레이어는 최대 3번의 시도 안에 정답을 맞춰야 하며, 승리자가 없을 경우 무승부로 종료됩니다.

## 프로젝트 개요
### 목표
- 네트워크 게임의 서버 권위(Server-authoritative) 구조 학습
- 플레이어 상태 관리(PlayerState)와 게임 진행(GameMode) 로직 분리
- 채팅 UI와 게임 로직 통합
### 게임규칙
- 서버가 게임 시작 시 중복되지 않는 3자리 숫자(1~9)를 생성합니다.
- 플레이어는 채팅창에 3자리 숫자를 입력 후 Enter 키로 전송합니다.
- 서버는 입력값을 검증하여 결과를 반환합니다:
    - Strike (S): 숫자와 위치 모두 일치
    - Ball (B): 숫자는 일치하지만 위치가 다름
    - Out: 0 Strike, 3 Ball일 경우
- 각 플레이어는 3번의 시도만 가능하며, 이를 초과하면 입력이 무효화됩니다.
- 누군가 3 Strike를 달성하면 즉시 승리자가 결정됩니다.
- 모든 플레이어가 시도를 소진했음에도 승리자가 없으면 무승부 처리됩니다.

## 구현 과정

### 1. 요구사항 분석
- 플레이어가 3번의 기회 안에 정답을 맞추는 숫자야구 게임
- 여러 명이 동시에 플레이할 수 있도록 **Dedicated Server** 구조 사용
- 채팅 시스템을 통해 입력 및 결과 전달

### 2. 설계
- **HMGameMode**: 정답 생성, 결과 판정, 게임 리셋
- **HMPlayerController**: 채팅 입력 및 서버와의 RPC 통신
- **HMPlayerState**: 플레이어 이름과 시도 횟수 관리
- **HMChatInput**: UMG 기반 채팅 입력창

### 3. 초기 세팅
- Unreal 프로젝트 생성 후 `HMChatting.Build.cs`에서 UMG 모듈 추가
- `HMGameModeBase`, `HMGameStateBase` 생성
- `HMPlayerController` 기본 뼈대 작성

### 4. 채팅 시스템 구현
- `UHMChatInput` 위젯을 생성하여 `EditableTextBox`로 입력 처리
- PlayerController → 서버 RPC 통신 → 서버에서 브로드캐스트

### 5. 숫자야구 게임 로직 구현
- **정답 생성**: `GenerateSecretNumber` 함수에서 1~9 숫자 중 3개 랜덤 선택
- **판정**: `JudgeResult` 함수로 Strike, Ball, Out 계산
- **시도 횟수 관리**: HMPlayerState의 `CurrentGuessCount`로 제한

### 6. 게임 흐름 완성
- 서버에서 입력을 받으면 판정 후 결과를 모든 클라이언트에게 전달
- 승리 조건: 3 Strike
- 무승부 조건: 모든 플레이어가 3회 소진

### 7. 테스트 및 디버깅
- 두 명의 클라이언트가 서버에 접속해 정상적으로 메시지가 전달되는지 확인
- 시도 횟수 제한 및 무승부 조건 확인
- 로그를 통해 RPC 호출 순서를 디버깅
